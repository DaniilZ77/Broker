# Broker

Брокер сообщений представляет собой приложение, внутри которого множество очередей. Очереди создаются при старте приложения через конфигурационный файл (во время исполнения приложения нельзя добавлять и удалять очереди). Каждая очередь фиксированного размера, а также с ней ассоциируется уникальное название, при помощи которого можно будет взаимодействовать с очередью (размер очереди и ключ также конфигуриуется при старте приложения). Сообщения в очередях хранятся в оперативной памяти. Сообщения могут представлять собой абсолютно любые данные, сериализованые в формате JSON. При отправке сообщения в очередь, сообщение будет доставлено всем подписчикам этой очереди. Если очередь переполнена, то брокер сообщений возвращает ошибку клиенту при отправке сообщения. Подписчиков у очереди ограниченное количество (ограничение конфигурируется при старте).

## API

```
POST ${address}/v1/queues/${queue_name}/messages
POST ${address}/v1/queues/${queue_name}/subscriptions
```
